## කොහොමද නිරීක්ෂණ කුළුණු ක්‍රියා කරන්නේ?


Lightning Network පරිසර පද්ධතියේ අත්‍යවශ්‍ය කොටසක් වන, කාවද්දම් මිනාරම් පරිශීලකයින්ගේ විදුලි නාලිකා සඳහා අමතර ආරක්ෂක මට්ටමක් ලබා දේ. එහි ප්‍රධාන වගකීම වන්නේ නාලිකා සෞඛ්‍යය පරීක්ෂා කිරීම සහ එක් නාලිකා පක්ෂයක් අනෙක් පක්ෂය වංචා කිරීමට උත්සාහ කරන විට මැදිහත් වීමයි.


Watchtower කෙසේ චැනලයක් අත්පත් කරගෙන ඇතිද යන්න තීරණය කළ හැකිද? Watchtower, චැනල් පක්ෂයන්ගෙන් එකක් වන ගනුදෙනුකරුවාගෙන් අවශ්‍ය තොරතුරු ලබා ගනී, ඕනෑම උල්ලංඝනයකට නිවැරදිව හඳුනාගෙන ප්‍රතිචාර දැක්වීම සඳහා. නවතම ගනුදෙනු විස්තර, වර්තමාන චැනල් තත්ත්වය, සහ දඩ ගනුදෙනු නිර්මාණය කිරීමට අවශ්‍ය තොරතුරු මෙම තොරතුරු වලට නිතරම ඇතුළත් වේ. දත්ත Watchtower වෙත සම්ප්‍රේෂණය කිරීමට පෙර, ගනුදෙනුකරු ඒවා සංකේතනය කළ හැකිය, රහස්‍යතාවය සහ රහස්‍යභාවය ආරක්ෂා කිරීම සඳහා. මෙය Watchtower වෙත දත්ත ලැබුණද, උල්ලංඝනයක් ඇත්තෙන්ම සිදු වී නැති නම් සංකේතනය කළ දත්ත විකේතනය කිරීමෙන් වළකයි. මෙම සංකේතන පද්ධතිය මගින් ගනුදෙනුකරුගේ රහස්‍යතාව ආරක්ෂා කරයි, එසේම අවසරයකින් තොරව පුද්ගලික දත්ත ප්‍රවේශ වීමෙන් Watchtower වළකයි.


## ඔබේම Satoshi ඇස පිහිටුවීම සහ දායක වීම ආරම්භ කිරීමේ ආකාරය


Satoshi කණ්ණාඩිය ([Rust-TEOS](https://github.com/talaia-labs/Rust-teos?ref=blog.summerofbitcoin.org)) යනු [Bolt 13](https://github.com/sr-gi/bolt13/blob/master/13-watchtowers.md?ref=blog.summerofbitcoin.org) සමඟ අනුකූල නොවන Lightning Watchtower එකකි. එය ප්‍රධාන කොටස් දෙකක් ඇත:


1. teos: CLI සහ ටවර්හි සේවාදායක-පාර්ශවීය මූලික ක්‍රියාකාරිතාව ඇතුළුව. මෙම පෙට්ටිය ගොඩනැගූ විට teosd සහ teos-CLI යන ද්විමය දෙකක් නිපදවනු ලැබේ.

2. teos-common: සමහර සේවාදායක පාර්ශවීය සහ සේවාලාභී පාර්ශවීය ක්‍රියාකාරිතාව (සේවාලාභී නිර්මාණය සඳහා උපකාරී).


ටවර්ය සාර්ථකව ක්‍රියාත්මක කිරීමට, teosd විධානය සමඟ ටවර්ය ක්‍රියාත්මක කිරීමට පෙර bitcoind ක්‍රියාත්මක විය යුතුය. මෙම විධාන දෙකම ක්‍රියාත්මක කිරීමට පෙර ඔබේ Bitcoin.conf ගොනුව සකස් කළ යුතුය. මෙය කෙසේ කරන්නද යන්න පිළිබඳ පියවර මෙන්න:-


1. Bitcoin මූලිකය මූලාශ්‍රයෙන් ස්ථාපනය කරන්න හෝ එය බාගන්න. බාගත කිරීමෙන් පසු, Bitcoin.conf ගොනුව Bitcoin මූලික පරිශීලක නාමාවලියට තබන්න. ගොනුව තැබිය යුතු ස්ථානය පිළිබඳ වැඩි විස්තර සඳහා මෙම සබැඳිය පරීක්ෂා කරන්න, එය ඔබ භාවිතා කරන මෙහෙයුම් පද්ධතිය මත රඳා පවතී.

2. फ़ाइल को बनाने के लिए स्थान की पहचान करने के बाद, इन विकल्पों को जोड़ें:-


```
#RPC
server=1
rpcuser=<your-user>
rpcpassword=<your-password>

#chain
regtest=1```

- server: For RPC requests
- rpcuser and rpcpassword: RPC clients can authenticate with the server
- regtest: Not required but useful if you're planning for development.

rpcuser and rpcpassword need to be picked by you. They must be written without quotes. Eg:-

```

rpcuser=aniketh

rpcpassword=strongpassword```


Zdaj, če zaženete bitcoind, bi moral začeti z zagonom vozlišča.


1. ටවර් කොටස සඳහා, පළමුව teos මූලාශ්‍රයෙන් ස්ථාපනය කළ යුතුය. මෙම සබැඳියෙහි දී ඇති උපදෙස් අනුගමනය කරන්න.

2. ඔබේ පද්ධතියේ teos සාර්ථකව ස්ථාපනය කිරීම සහ පරීක්ෂණ ධාවනය කිරීමෙන් පසු, ඔබට අවසාන පියවර වන teos.toml ගොනුව teos පරිශීලක නාමාවලියට පිහිටුවිය හැක. ගොනුව .teos (ඉරියව්වට අවධානය යොමු කරන්න) නම් කරන ලද ෆෝල්ඩරයක ඔබේ මුල් ෆෝල්ඩරය යටතේ තැබිය යුතුය. එනම්, උදාහරණයක් ලෙස, Linux සඳහා /home/<your-username>/.teos. ඔබ ස්ථානය සොයාගත් පසු, teos.toml ගොනුවක් සාදන්න සහ අපි bitcoind මත වෙනස් කළ දේවල් වලට අදාළව මෙම විකල්ප සකසන්න.


```
#bitcoind
btc_network = "regtest"
btc_rpc_user = <your-user>
btc_rpc_password = <your-password>```

Notice that here the username and password need to be written quoted, that is, for the same example as before:

```

btc_rpc_user = "aniketh"

btc_rpc_password = "strongpassword"```

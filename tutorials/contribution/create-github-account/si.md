Plan ₿'s mission is to provide top-tier educational resources about Bitcoin, available in as many languages as possible. All content published on the site is open-source and hosted on GitHub, offering anyone the opportunity to contribute to the enrichment of the platform. Contributions can take various forms: correcting and proofreading existing texts, translations into other languages, updating information, or creating new tutorials not yet available on our site.


ඔබ Plan ₿ Network සඳහා දායක වීමට කැමති නම්, ඔබ Git සහ GitHub භාවිතා කළ යුතු වන අතර, මෙම මෙවලම් ඔබට අමුතු නම් හෝ එහි ක්‍රියාකාරීත්වය අපැහැදිලි නම්, බිය නොවන්න, මෙම ලිපිය ඔබ සඳහායි! අපි Git සහ GitHub හි මූලික කරුණු, මෙන්ම සම්බන්ධ තාක්ෂණික භාෂාව, එකට සමාලෝචනය කරමු, එවිට ඔබට පසුව එය ප්‍රභලව භාවිතා කළ හැකිය.


## Git kaj je?


Git යනු අනුවාද පාලන පද්ධතියක් වන අතර, විශේෂයෙන්ම මෘදුකාංග ව්‍යාපෘති කළමනාකරණය කිරීම සඳහා නිර්මාණය කරන ලද්දකි. 2005 දී Linus Torvalds විසින් නිර්මාණය කරන ලද Git, අනුවාද පාලනය සඳහා මෘදුකාංග සංවර්ධන කර්මාන්තයේ සම්මතය බවට ඉක්මනින් පත්වී ඇත. නමුත් එය නිවැරදිව කුමක්ද?

![git](assets/11.webp)

එහි මූලිකත්වයෙහි, Git සංවර්ධකයින්ට ව්‍යාපෘතියක මූලාශ්‍ර කේතයට කාලයත් සමඟ සිදු කරන වෙනස්කම් හඹා යාමට ඉඩ සලසයි. මෙය අර්ථ දක්වන්නේ කේතයේ සෑම වෙනස්කමක් සමඟම, Git ව්‍යාපෘතියේ නව අනුවාදයක් සටහන් කරයි. දෝෂයක් සිදුවන්නේ නම් හෝ පරීක්ෂණාත්මක විශේෂාංගයක් අපේක්ෂිත ලෙස ක්‍රියා නොකරන්නේ නම්, ගොනු සඳහා කාල යන්ත්‍රයක් වැනි, කේතයේ පෙර අවස්ථාවකට ආපසු යාමට හැකියාව ඇත.


Git හි ප්‍රධාන ලක්ෂණ වලින් එකක් ශාඛා කළමනාකරණය වේ. ශාඛාව යනු සංවර්ධකයින්ට ව්‍යාපෘතියේ අනෙක් කොටස් වලින් ස්වාධීනව වැඩ කිරීමට හැකි සමාන්තර රේඛාවක් වැනි එකකි. මෙය නව ලක්ෂණ එකතු කිරීම හෝ දෝෂ නිවැරදි කිරීම ප්‍රධාන කේතය බාධා නොකිරීමෙන් ඉතා පහසු කරයි. වෙනස්කම් පරීක්ෂා කර සනාථ කළ පසු, ඒවා ප්‍රධාන ශාඛාව සමඟ ඒකාබද්ධ කළ හැක.


Git හි විශේෂත්වයන්ගෙන් එකක් වන්නේ එහි බෙදා හැරීමේ ක්‍රමයකින් ක්‍රියා කිරීමට ඇති හැකියාවයි. සෑම සංවර්ධකයෙකුටම Hard ධාවකයෙහි තමන්ගේම පරිගණකයේ ව්‍යාපෘතියේ සම්පූර්ණ පිටපතක් තිබේ, එමඟින් ඔවුන්ට අන්තර්ජාල සම්බන්ධතාවයක් ලබා ගත හැකි විට වෙනස්කම් එකතු කරමින් අන්තර්ජාල සම්බන්ධතාවයක් නොමැතිව වැඩ කිරීමට හැකියාව ලැබේ. මෙය ගැටුම් ඇති වීමේ අවදානම අඩු කරයි සහ එකම ව්‍යාපෘතියේ එකම වේලාවේදී බහු සංවර්ධකයින්ට එකම ව්‍යාපෘතියේ වැඩ කිරීමට ඉඩ සලසයි.

මුලදී, Git මෘදුකාංග සංවර්ධන ව්‍යාපෘති සඳහා ප්‍රධාන වශයෙන් නිර්මාණය කරන ලදී. කෙසේ වෙතත්, එය අන්තර්ගත ලිවීමේ ව්‍යාපෘති කළමනාකරණය කිරීමට ද භාවිතා කළ හැක. කේතය මත සහයෝගයෙන් කටයුතු කිරීම වෙනුවට, අපි පෙළ මත සහයෝගයෙන් කටයුතු කරමු. සහ Plan ₿ Network විසින් එහි අන්තර්ගතය කළමනාකරණය කිරීමට අනුගමනය කර ඇති ක්‍රමය හුදෙක් මෙයයි! Git ලිවීමේ පාඨමාලා සහ උපකාරක පන්ති මත සහයෝගයෙන් කටයුතු කිරීම පහසු කරයි, මන්ද එය වෙනස්කම් නිවැරදිව හඹා යාම, කාර්යක්ෂම අනුවාද කළමනාකරණය, සහ අනෙක් දායකයින් විසින් අන්තර්ගතය සමාලෝචනය කිරීම සහ වැඩිදියුණු කිරීම සඳහා ද හැකියාව සපයයි.


## GitHub කුමක්ද?


GitHub යනු අපි මේ දක්වා සාකච්ඡා කළ Git අනුවාද පාලන පද්ධතිය මත පදනම් වූ මූලාශ්‍ර කේත කළමනාකරණ සහ සත්කාරක වේදිකාවකි. 2008 දී ආරම්භ කරන ලද GitHub ඉක්මනින් ජනප්‍රියත්වයට පත්විය අතර ලොව පුරා සංවර්ධකයින් සඳහා අත්‍යවශ්‍ය යොමුකිරීමක් වී ඇත. නමුත් GitHub Git වෙතින් කෙසේ වෙන්වෙයිද, සහ අපගේ අන්තර්ගත නිෂ්පාදන ක්‍රමය තුළ එය එතරම් වැදගත් වන්නේ මන්ද?

![github](assets/12.webp)

පළමුව, GitHub Git මත නිර්මාණය වී ඇති බව තේරුම් ගැනීම වැදගත් (අපි පෙර කොටසෙහි සාකච්ඡා කළා). Git කේත වෙනස්කම් හඹා යන මෙවලම වන අතර, GitHub මෙම කේතය සත්කාරක, බෙදා හදා ගැනීම සහ කළමනාකරණය කරන මාර්ගගත සේවාවයි.


Git යනු සෑම සංවර්ධකයෙකුම තමන්ගේ පරිගණකයේ භාවිතා කරන ලොග් පොතක් වැනි එකක් ලෙස කල්පනා කරන්න, එය ඔවුන්ගේ ව්‍යාපෘතියේ සියලු වෙනස්කම් සටහන් කිරීමට භාවිතා කරයි. GitHub, අනෙක් අතට, මෙම සියලුම ලොග් පොත් බෙදාගත හැකි, සසඳා බැලිය හැකි, සහ එකට එක් කළ හැකි මහජන පුස්තකාලයක් වැනි එකක්.

## සමහර තාක්ෂණික පද


Git සහ GitHub මත, ඔබට නාමයන් සංකීර්ණ ලෙස පෙනෙන විධාන සහ විශේෂාංග හමුවිය හැක. මෙම අවසන් කොටසෙහි, Git සහ GitHub භාවිතා කිරීමේදී ඔබට හමුවිය හැකි තාක්ෂණික පද සරලව තේරුම් ගැනීමට කිහිපයක් සරල නිර්වචන ලබා දෙනු ඇත:
